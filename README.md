# 🐭 囤囤鼠历险记 - 自动游戏 Bot

一个用于「囤囤鼠历险记」浏览器游戏的自动化工具，通过 WebSocket 连接游戏服务器，自动收集金币、躲避敌人、管理体力。

## ✨ 功能特性

- **自动收集金币** — 智能寻找金币密集区域，优先收集高价值金币
- **自动躲避敌人** — 普通玩家保持 200m 距离，危险杀手保持 300m 距离
- **体力管理** — 自动监控三个时间窗口的体力，耗尽时自动下线休息
- **蹲守者追踪** — 记录攻击者，递增休息时长（5→10→20 分钟）
- **杀手识别** — 追踪击杀 2 人以上的玩家，保持更远距离
- **Web 控制面板** — 实时查看游戏状态、日志、小地图
- **Token 持久化** — 首次登录后 Token 自动保存，之后只需点击上线/下线

## 🚀 使用方法

### 启动 Bot

```bash
npm start
```

启动后访问 http://localhost:38473 控制面板。

### 首次登录

**方式一：通过控制面板手动输入**

1. 打开 http://localhost:38473
2. 在「首次登录」区域输入 User ID 和 Session Token
3. 点击「保存并启动」

**方式二：通过 Chrome 扩展自动获取**

1. 安装 Chrome 扩展（见上方）
2. 访问 https://grasp-rat-game.h-e.top 并登录
3. 扩展会自动检测并发送 Token 到 Bot
4. 或者点击扩展图标 → 点击「发送 Token」按钮

### 获取 Token 的方法

如果不想安装扩展，可以手动获取：

1. 打开游戏网站并登录
2. 按 `F12` 打开开发者工具
3. 在 Console 中输入：
   ```javascript
   localStorage.tmpGameUserId      // 获取用户 ID
   localStorage.tmpGameSessionToken // 获取 Session Token
   ```

### 日常使用

Token 保存后，只需访问控制面板：

- **🟢 上线** — 启动 Bot，开始自动游戏
- **🔴 下线** — 停止 Bot，断开游戏连接

## 🎮 策略详解

### 核心策略模式

Bot 有三种主要运行模式：

#### 1. 收集模式 (collect)

**触发条件**：没有危险敌人，体力充足

**行为**：
- 寻找金币密集区域（150m 范围内金币最多的位置）
- 优先选择安全的金币（避开 200m 内有玩家的金币）
- 远距离（>500m）使用传送技能
- 持续移动收集金币

**评分算法**：
```
得分 = 金币数量 × 10 - 距离(m) / 100
```

#### 2. 逃跑模式 (flee)

**触发条件**：检测到危险敌人

**行为**：
- 立即远离威胁方向
- 被追超过 30 秒 → 强制下线休息
- 逃跑期间不收集金币

**危险判定**：
- 普通玩家：200m 范围内
- 危险杀手（击杀 2+）：300m 范围内

#### 3. 休息模式 (rest)

**触发条件**：体力耗尽或被攻击

**行为**：
- 停止所有游戏操作
- 调用游戏 API 离开游戏
- 断开 WebSocket 连接
- 定时检查体力恢复情况
- 恢复后自动重新上线

### 体力系统

游戏有三个时间窗口的体力限制：

| 窗口 | 上限 | 恢复时间 | 说明 |
|------|------|----------|------|
| 5 秒 | 10s | 自动恢复 | 短期体力，影响射击 |
| 1 小时 | 3000s | 1 小时 | 小时限额 |
| 1 天 | 20000s | 24 小时 | 日限额 |

**体力消耗**：
- 移动：每 10m 消耗 1 单位
- 射击：每发消耗 500ms
- 传送：消耗 1500000 单位（1h 和 1d 窗口）

**休息策略**：
- 任一窗口耗尽 → 自动下线
- 每 5 分钟检查一次体力恢复
- 恢复后自动重新上线

### 蹲守者追踪

当被其他玩家攻击时，Bot 会：

1. **记录攻击者** — 记录攻击者 ID 和攻击次数
2. **递增休息时长**：
   - 第 1 次被攻击：休息 5 分钟
   - 第 2 次被攻击：休息 10 分钟
   - 第 3 次及以上：休息 20 分钟
3. **保留记录** — 休息恢复后保留蹲守者记录，避免重复被蹲

### 杀手识别

Bot 会追踪游戏中的击杀事件：

- **记录杀手** — 从游戏消息中提取击杀者信息
- **危险判定** — 击杀 2 人以上的玩家视为危险杀手
- **更远距离** — 对危险杀手保持 300m 距离（普通玩家 200m）
- **优先逃跑** — 检测到危险杀手时优先逃跑

### 危险区域判定

Bot 会动态计算危险区域：

```
普通玩家：200m 范围内 → 触发逃跑
危险杀手：300m 范围内 → 触发逃跑
金币安全区：金币 200m 范围内无玩家才收集
```

## ⚙️ 配置说明

### 关键常量

在 `game-state.js` 中定义：

```javascript
CONSTANTS = {
  WORLD_RADIUS_CM: 1000000,        // 世界半径 10km
  SERVER_TICK_MS: 50,              // 服务器 tick 间隔 50ms
  PLAYER_SPEED_PER_TICK: 50,      // 玩家速度
  BULLET_RANGE_CM: 15000,         // 射程 150m
  BULLET_DAMAGE: 3,               // 每发伤害
  TELEPORT_STAMINA_COST: 1500000, // 传送体力消耗
}
```

### 策略参数

在 `strategies.js` 中定义：

```javascript
DANGER_RADIUS = 20000;           // 普通玩家危险距离 200m
KILLER_DANGER_RADIUS = 30000;   // 杀手危险距离 300m
COIN_CLUSTER_RADIUS = 15000;    // 金币聚集判定范围 150m
CHASE_TIMEOUT_TICKS = 600;      // 被追超时 30 秒
```

### 端口配置

在 `index.js` 中定义：

```javascript
LOCAL_PORT = 38472;     // Token 接收服务器
DASHBOARD_PORT = 38473; // Web 控制面板
```

## 🔧 API 接口

| 端点 | 方法 | 说明 |
|------|------|------|
| `/token` | POST | 接收 Token 并启动 Bot |

### Dashboard 服务器 (端口 38473)

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/state` | GET | 获取游戏状态 |

## 🐛 常见问题

### Q: Bot 启动后无法连接游戏

**可能原因**：
1. Token 过期 — 重新获取 Token
2. 网络问题 — 检查网络连接
3. 游戏服务器维护 — 等待维护结束

**解决方法**：
- 重新登录游戏网站获取新 Token
- 通过控制面板重新输入 Token

### Q: Bot 频繁下线休息

**可能原因**：
1. 体力耗尽 — 正常行为，等待恢复
2. 被其他玩家攻击 — 蹲守者追踪机制
3. 被追杀超过 30 秒 — 强制下线保护

**解决方法**：
- 体力耗尽时耐心等待恢复
- 如果被频繁蹲守，可以手动下线一段时间后再上线

### Q: 控制面板无法访问

**可能原因**：
1. Bot 未启动 — 运行 `npm start`
2. 端口被占用 — 检查端口 38472 和 38473
3. 防火墙阻止 — 检查防火墙设置

**解决方法**：
- 确保 Bot 正在运行
- 尝试重启 Bot
- 检查端口占用情况

### Q: Chrome 扩展无法发送 Token

**可能原因**：
1. 扩展未安装 — 按照说明安装扩展
2. Bot 服务器未启动 — 先运行 `npm start`
3. 权限问题 — 检查扩展权限设置

**解决方法**：
- 确保扩展已正确安装
- 确保 Bot 正在运行
- 尝试手动输入 Token

- **体力 5s** — 短期体力

- **危险杀手** — 击杀 2+ 的玩家列表

- **🔵 蓝色圆点** — 你的位置

- **红色虚线圈** — 200m 危险区域
- **橙色虚线圈** — 300m 杀手危险区域
